# SMCP Trainer Master Blueprint

## Platform Identity

SMCP Trainer is a professional maritime immersion simulator.

It is not an English course.

It is not a classroom exercise platform.

It trains cadets to perform real shipboard communication tasks under operational pressure using STCW competencies and IMO Standard Marine Communication Phrases.

The purpose is operational readiness.

Every module, mission, simulation, assessment, and career gate must connect to:

- STCW competency
- IMO SMCP function
- Real shipboard task
- Mission-based assessment

The platform must train cadets to observe, decide, communicate, report, confirm, and act as maritime professionals.

---

## Core Design Rules

The platform must not use traditional ESL training patterns.

Do not include:

- Vocabulary drills
- Basic quizzes
- Listen-and-repeat exercises
- Grammar lessons
- Fill-in-the-blank activities
- Generic beginner dialogues
- Classroom-style pronunciation practice
- Isolated phrase memorization
- Non-operational language tasks

All learning happens through operational missions.

The cadet must use maritime language because the shipboard situation requires it.

The cadet must not study phrases in isolation. The cadet must use them to complete shipboard tasks.

---

## Required Activity Types

Old activity types are replaced with professional simulator activities:

- Equipment Recognition
- Situation Reports
- Watchkeeping Missions
- Emergency Response Missions
- Bridge/Engine Communication
- Weather and Sea-State Decisions
- Pollution and Security Incident Reports
- Final Checkrides

Each activity must include:

- Operational setting
- Assigned role
- Task objective
- Required communication function
- STCW competency reference
- IMO SMCP function reference
- Performance standard
- Assessment result

# 1. Deck Academy

## Purpose

Deck Academy trains cadets for professional bridge, deck, cargo, navigation, radio, safety, environmental, and security communication tasks.

The Deck pathway prepares cadets for watchkeeping support and future Officer of the Watch responsibilities.

## Core Competency Sources

Deck Academy is aligned with:

- STCW Table A-II/1
- OICNW competency standards
- IMO SMCP external communication
- IMO SMCP onboard communication
- Bridge watchkeeping procedures
- Radio communication procedures
- Emergency response procedures
- Pollution prevention reporting
- Maritime security reporting

## Primary Reference Resource Areas

Deck Academy should draw source material from:

- Maritime-English Elementary: The Navigation Bridge
- Maritime-English Elementary: Radio Communication
- Maritime-English Elementary: The Weather
- Maritime-English Elementary: Severe Weather Conditions
- Maritime-English Elementary: Cargo Operations
- Maritime-English Elementary: The Marine Environment
- Maritime-English Intermediate: SMCP Environmental Protection Communications
- Maritime-English Intermediate: Reporting Pollution Incidents
- Maritime-English Maritime Security modules
- SMCP Resolution A.918(22)

## Deck Academy Structure

### Deck Level 1: Cadet Shipboard Orientation

Operational focus:

- Shipboard chain of command
- Deck department responsibilities
- Bridge access and conduct
- Safety equipment awareness
- Basic operational reporting

Mission types:

- Equipment Recognition
- Situation Reports
- Bridge/Engine Communication

Representative missions:

- Report to the Bridge
- Identify Bridge Watch Stations
- Confirm Deck Assignment
- Inspect Personal Safety Equipment
- Report Unsafe Deck Condition

Assessment standard:

The cadet can receive an operational assignment, confirm the task, proceed to the correct station, and report status clearly.

### Deck Level 2: Bridge Familiarization and Watch Support

Operational focus:

- Bridge equipment recognition
- Lookout duties
- Contact reporting
- Course, speed, heading, position language
- Watch support communication

Mission types:

- Equipment Recognition
- Watchkeeping Missions
- Situation Reports

Representative missions:

- Identify Bridge Equipment
- Maintain Lookout
- Report Contact on Starboard Side
- Report Vessel Ahead
- Complete Watch Support Log

Assessment standard:

The cadet can identify bridge equipment, maintain lookout, report observed contacts, and communicate without ambiguity.

### Deck Level 3: Radio and SMCP Operations

Operational focus:

- VHF communication
- Message markers
- Port control communication
- Pilot station communication
- Tug and berthing communication
- Emergency radio calls

Mission types:

- Bridge/Engine Communication
- Situation Reports
- Emergency Response Missions

Representative missions:

- Call Port Control
- Request Pilot Boarding
- Confirm Tug Requirement
- Report Arrival Status
- Send Distress Communication

Assessment standard:

The cadet can use IMO SMCP functions to send, receive, confirm, and clarify operational messages.

### Deck Level 4: Weather, Sea State, and Navigation Decisions

Operational focus:

- Weather observation
- Visibility report
- Wind and sea-state report
- Heavy-weather preparation
- Safe speed decision support
- Restricted visibility communication

Mission types:

- Weather and Sea-State Decisions
- Watchkeeping Missions
- Situation Reports

Representative missions:

- Report Weather Deterioration
- Report Reduced Visibility
- Recommend Heavy-Weather Deck Preparation
- Confirm Safe Speed Advisory
- Report Vessel Motion

Assessment standard:

The cadet can observe weather and sea-state conditions, report them accurately, and support safe bridge decision-making.

### Deck Level 5: Cargo, Environment, and Security Operations

Operational focus:

- Cargo area recognition
- Dangerous goods reporting
- Cargo leak reporting
- Oil spill reporting
- Security patrol reporting
- Restricted area control

Mission types:

- Equipment Recognition
- Pollution and Security Incident Reports
- Emergency Response Missions

Representative missions:

- Inspect Cargo Area
- Report Dangerous Goods Condition
- Report Oil Spill on Deck
- Challenge Unauthorized Person
- Report Security Breach

Assessment standard:

The cadet can report cargo, pollution, and security incidents using operationally useful information and appropriate SMCP functions.

### Deck Final Checkride

Scenario:

A full bridge watch simulation from pre-departure to restricted visibility and port arrival.

Required tasks:

- Receive watch briefing
- Confirm bridge assignment
- Identify active bridge equipment
- Maintain lookout
- Report traffic contact
- Report weather change
- Communicate with port control
- Respond to emergency message
- Complete watch handover

Required assessment links:

- STCW: Maintain a safe navigational watch
- STCW: Use English in written and oral form
- STCW: Respond to emergencies
- IMO SMCP: External communication
- IMO SMCP: Onboard communication
- Real task: Bridge watch support

Passing standard:

The cadet completes the watch with no critical communication failure, no missed emergency report, and acceptable operational clarity.

# 3. Ocean Intelligence Center

## Purpose

The Ocean Intelligence Center is the operational knowledge and decision-support hub of the platform.

It is not a library page.

It is an intelligence workspace where cadets review shipboard references, operational notices, competency maps, mission records, and scenario briefings.

## Core Functions

The Ocean Intelligence Center should provide:

- STCW competency map
- IMO SMCP function map
- Mission intelligence briefs
- Vessel and department profiles
- Equipment recognition archive
- Incident report archive
- Weather and sea-state reference board
- Pollution and security reporting board
- Career progression record
- Final checkride readiness review

## Intelligence Records

Each mission should produce an intelligence record:

- Mission ID
- Cadet role
- Vessel
- Department
- Location
- Task performed
- STCW competency practiced
- SMCP function used
- Critical decision points
- Communication errors
- Operational outcome
- XP awarded
- Readiness recommendation

## STCW Competency Map

The platform should map missions to competency groups.

Deck competency groups:

- Plan and conduct a passage and determine position
- Maintain a safe navigational watch
- Use radar and ARPA to maintain safety of navigation
- Use ECDIS to maintain safety of navigation
- Respond to emergencies
- Respond to distress signals at sea
- Use English in written and oral form
- Transmit and receive information by visual signaling

Engine competency groups:

- Maintain safe engineering watch
- Operate main and auxiliary machinery
- Monitor machinery condition
- Respond to machinery emergencies
- Prevent pollution
- Apply occupational safety procedures
- Use English in written and oral form
- Communicate with bridge and engineering team

## IMO SMCP Function Map

SMCP functions should be organized by operational use, not by textbook order.

Recommended function groups:

- Distress communication
- Urgency communication
- Safety communication
- Message markers
- Repetition and confirmation
- Position reporting
- Course and speed communication
- Traffic and collision risk reporting
- Pilotage communication
- Tug and berthing communication
- VTS communication
- Weather reporting
- Emergency onboard communication
- Fire and damage control communication
- Medical assistance communication
- Pollution reporting
- Security reporting
- Bridge-engine communication

## Intelligence Center User Flow

1. Cadet opens active mission brief.
2. Cadet reviews vessel, department, location, and risk factors.
3. Cadet sees required STCW competency and SMCP function.
4. Cadet enters Operations Simulator.
5. Simulator records performance.
6. Ocean Intelligence Center updates readiness status.
7. Cadet receives next recommended mission.

# 4. Operations Simulator

## Purpose

The Operations Simulator is the core interactive training environment.

It should simulate shipboard work, not classroom practice.

Cadets must complete operational tasks under mission conditions.

## Simulator Modes

### Equipment Recognition

Purpose:

Train cadets to identify equipment that matters for the current mission.

Examples:

- Bridge equipment
- Engine-room machinery
- Safety equipment
- Cargo equipment
- Pollution response equipment
- Security access points

Assessment:

The cadet must identify correct equipment and connect it to the assigned task.

Failure examples:

- Wrong equipment selected
- Critical safety item missed
- Incorrect location identified
- Equipment function misunderstood

### Situation Reports

Purpose:

Train cadets to report operational status clearly.

Examples:

- Contact on starboard side
- Weather deteriorating
- Pump pressure abnormal
- Cargo leak observed
- Unauthorized person near restricted area

Assessment:

The cadet must report:

- What happened
- Where it happened
- Current status
- Required action or assistance

### Watchkeeping Missions

Purpose:

Train cadets to support bridge or engine-room watches.

Deck examples:

- Maintain lookout
- Report contact
- Monitor visibility
- Support watch handover

Engine examples:

- Complete machinery round
- Check readings
- Report abnormal sound
- Complete watch handover

Assessment:

The cadet must observe, decide, communicate, and hand over accurately.

### Emergency Response Missions

Purpose:

Train cadets to communicate during urgent or dangerous conditions.

Examples:

- Man overboard
- Fire in engine room
- Main engine failure
- Flooding
- Medical emergency
- Distress call

Assessment:

The cadet must identify emergency type, report location, request assistance, and follow chain of command.

### Bridge/Engine Communication

Purpose:

Train operational communication between deck and engine departments.

Examples:

- Stand by engine
- Main engine ready
- Confirm speed order
- Report machinery limitation
- Request bridge confirmation

Assessment:

The cadet must confirm orders, report status, and avoid ambiguous communication.

### Weather and Sea-State Decisions

Purpose:

Train cadets to observe and report weather and sea-state conditions that affect safety.

Examples:

- Reduced visibility
- Heavy swell
- Strong wind
- Vessel rolling
- Deck unsafe for work

Assessment:

The cadet must report conditions and support safe operational decisions.

### Pollution and Security Incident Reports

Purpose:

Train cadets to escalate pollution and security incidents correctly.

Examples:

- Oil spill on deck
- Fuel leak in engine room
- Bilge alarm
- Unauthorized access
- Suspicious object
- Security breach

Assessment:

The cadet must report incident type, location, risk, immediate action, and required support.

### Final Checkrides

Purpose:

Assess operational readiness through multi-stage simulation.

Deck final checkride:

- Bridge watch from briefing to handover
- Radio communication
- Weather report
- Emergency response
- Port arrival communication

Engine final checkride:

- Engine watch from briefing to handover
- Machinery readings
- Bridge-engine communication
- Alarm response
- Emergency escalation

Assessment:

The cadet must complete the scenario with no critical operational communication failure.

# 5. Career Progression System

## Purpose

The Career Progression System converts mission performance into maritime readiness status.

Progression is based on:

- XP
- Mission completion
- STCW competency coverage
- SMCP function coverage
- Checkride performance
- Critical error history

XP alone must not unlock professional status.

Professional status requires demonstrated mission performance.

## Career Tracks

### Deck Track

Progression target:

Officer of the Watch Candidate

Final readiness title:

Officer of the Watch

### Engine Track

Progression target:

Engineering Watchkeeping Candidate

Final readiness title:

Engineering Watch Officer

## Recommended XP Progression

### Cadet Recruit

Required XP:
0

Operational meaning:

The cadet has entered the academy but has not demonstrated shipboard readiness.

Required gates:

- Select department
- Complete first operational briefing

### Junior Cadet

Required XP:
250

Operational meaning:

The cadet can follow simple shipboard assignments and report to the correct station.

Required gates:

- Complete first Equipment Recognition mission
- Complete first Situation Report
- No critical chain-of-command failure

### Cadet Trainee

Required XP:
750

Operational meaning:

The cadet can confirm orders, complete supervised tasks, and report normal status.

Required gates:

- Complete department orientation missions
- Use confirmation and repetition functions correctly
- Complete one supervised operational task

### Department Cadet

Required XP:
1500

Operational meaning:

The cadet can operate inside a department mission sequence.

Required gates:

- Complete five department-specific missions
- Complete one safety-related mission
- Complete one equipment recognition assessment

### Watch Assistant

Required XP:
2500

Operational meaning:

The cadet can support bridge or engine watch under supervision.

Required gates:

- Complete first Watchkeeping Mission
- Submit acceptable watch status report
- Demonstrate correct handover communication

### Senior Cadet

Required XP:
4000

Operational meaning:

The cadet can complete routine department tasks with reduced supervision.

Required gates:

- Complete ten department missions
- Complete one communication mission
- Complete one abnormal condition report

### Watchkeeping Trainee

Required XP:
5500

Operational meaning:

The cadet can support operational decisions during changing conditions.

Required gates:

- Complete weather or machinery abnormality mission
- Complete bridge-engine or department communication mission
- Complete one pollution or security incident report

### OOW Candidate or Engineering Watchkeeping Candidate

Required XP:
7000

Operational meaning:

The cadet has completed the main academy path and is preparing for final checkride.

Required gates:

- Complete all required Level 1-4 missions
- Cover required STCW competency set
- Cover required SMCP function set
- No unresolved critical performance failure

### Officer of the Watch or Engineering Watch Officer

Required XP:
9000

Operational meaning:

The cadet has passed final simulator assessment.

Required gates:

- Complete final checkride
- Score at least 85 percent
- No missed emergency escalation
- No critical communication ambiguity
- Complete mission debrief

## XP by Mission Type

Equipment Recognition:
50 XP

Situation Report:
75 XP

Bridge/Engine Communication:
100 XP

Watchkeeping Mission:
150 XP

Weather and Sea-State Decision:
175 XP

Pollution or Security Incident Report:
200 XP

Emergency Response Mission:
250 XP

Final Checkride:
500 XP

## Critical Error Rules

Critical errors should block promotion until remediated through another mission.

Critical errors include:

- Failure to report emergency
- Incorrect emergency location
- Missed alarm
- Ambiguous bridge-engine order confirmation
- Unsafe weather decision
- Failure to report pollution incident
- Failure to report security breach
- Wrong department or officer notified
- Incomplete watch handover

## Mission Debrief Requirements

Every mission must end with an operational debrief.

Debrief must include:

- Mission outcome
- STCW competency practiced
- IMO SMCP function used
- Real shipboard task completed
- Critical decisions made
- Communication performance
- Errors and consequences
- XP awarded
- Readiness impact
- Next assigned mission

# Platform Architecture Recommendation

## Primary Systems

The platform should be organized around five systems:

1. Deck Academy
2. Engine Academy
3. Ocean Intelligence Center
4. Operations Simulator
5. Career Progression System

## Content Model

Each academy item should be stored as structured mission data.

Recommended structure:

```text
academy/
  master-blueprint.md
  reference/
  missions/
    deck/
    engine/
    shared/
  checkrides/
    deck/
    engine/
  intelligence/
    stcw-map.md
    smcp-function-map.md
    source-resource-map.md
```

## Mission Data Fields

Each mission should define:

- Mission ID
- Mission title
- Academy track
- Cadet role
- Vessel
- Department
- Location
- Operational context
- Real shipboard task
- STCW competency
- IMO SMCP function
- Required observations
- Required decisions
- Required communication
- Equipment involved
- Incident type if applicable
- Assessment criteria
- Critical errors
- XP reward
- Unlock conditions
- Debrief output

## Source Resource Use

The Maritime-English and SMCP folders should be treated as source libraries.

They should not dictate course structure.

Use them to extract:

- Operational scenarios
- Equipment images
- Maritime audio assets
- Radio communication examples
- SMCP functions
- Emergency examples
- Weather and sea-state references
- Cargo, security, and pollution scenarios
- Engine-room and bridge context

The academy structure must remain simulator-first.

# MVP Blueprint

## MVP Goal

Build a playable professional simulator path from Cadet Recruit to Watch Assistant.

## MVP Required Systems

- Track selection: Deck or Engine
- Mission selector
- Operations Simulator
- Ocean Intelligence Center mission brief
- XP and rank progression
- Mission debrief
- STCW and SMCP mapping

## MVP Deck Missions

1. Report to the Bridge
2. Identify Bridge Equipment
3. Maintain Lookout
4. Report Contact on Starboard Side
5. Call Port Control
6. Report Weather Deterioration
7. Respond to Man Overboard
8. Deck Watch Handover Checkride

## MVP Engine Missions

1. Report to Engine Control Room
2. Identify Engine-Room Equipment
3. Complete Engine-Room Round
4. Report Abnormal Pressure
5. Confirm Stand By Engine
6. Report Main Engine Ready
7. Respond to Engine-Room Fire
8. Engine Watch Handover Checkride

## MVP Shared Missions

1. Safety Equipment Inspection
2. Emergency Alarm Response
3. Pollution Incident Report
4. Security Breach Report

## MVP Passing Standard

The MVP is complete when a cadet can:

- Select Deck or Engine Academy
- Receive a professional mission brief
- Complete operational simulator tasks
- Use required SMCP functions during mission tasks
- Receive mission debrief
- Earn XP
- Advance from Cadet Recruit to Watch Assistant
- Complete one department checkride

# Long-Term Vision

SMCP Trainer should become a mission-based maritime readiness platform.

The final product should feel like entering a vessel training simulator, not opening a language textbook.

The cadet should progress by proving operational performance:

- Recognize equipment
- Understand the situation
- Make a safe decision
- Communicate using IMO SMCP
- Complete the shipboard task
- Receive professional debrief
- Advance through career readiness gates

The standard is not language knowledge.

The standard is safe maritime operation through clear professional communication.
