# SMCP Trainer Task Standard v2

SMCP Trainer is a Digital Maritime Academy. It is not an English course.

This standard defines how future academy tasks must be written when transforming official PN and MN academic programs into student missions.

Official methodology:

```text
Program -> Unit -> Topic -> Task -> Assessment
```

This document is a quality standard only. It does not generate tasks, modify the pilot, or replace the academic inventory.

## Purpose

Task Standard v2 exists to prevent generic, repeated task templates.

Every task must preserve academic traceability while feeling like an operational maritime cadet mission. The learner should not simply answer language or theory prompts. The learner should perform a realistic action, make a decision, report information, calculate something useful, identify a correct technical element, or complete a practical maritime workflow.

## Official Task Structure

Every high-quality SMCP Trainer task must use the following structure.

### Source Program

The official academic program the task comes from.

Examples:

- Piloto Naval
- Maquinista Naval

### Source Track

The academy track associated with the program.

Examples:

- Deck Academy
- Engine Academy

### Source Semester

The official semester where the subject appears.

### Source Subject

The official subject name from the academic inventory.

### Source Unit

The official unit from the academic program.

### Source Topic

The single official source topic that justifies the task.

Each task must trace back to exactly one source topic. A task may reference supporting knowledge, but its academic origin must remain clear.

### Mission Title

A mission-style title that tells the student what they will do.

Rules:

- Do not use titles such as "Practice 1.1".
- Do not use generic classroom labels.
- Use operational mission names.
- Prefer action verbs.

Examples:

- Mission 1.1: Report To The Chief Engineer
- Mission 2.3: Plot Vessel Position On A Nautical Chart
- Mission 4.2: Calculate Cargo Storage Dimensions

### Mission Brief

A short explanation of the mission context and why the task matters onboard.

The brief should connect the source topic to a maritime training situation without inventing new academic content.

### Operational Scenario

A specific onboard or port-related situation where the student must act.

The scenario must not be a reusable template with only the topic name changed. It should include enough maritime context to make the mission feel real.

### Cadet Role

The role the student performs during the mission.

Examples:

- Engine cadet reporting to the chief engineer
- Deck cadet preparing chart work
- Cadet assisting with cargo calculations
- Cadet identifying machinery drawing views

### Operational Objective

The action the student must complete successfully.

The objective must be performance-based. Grammar, vocabulary, or theory may support the action, but they must not be the final objective.

### Key Vocabulary

A short list of operational terms needed to complete the mission.

Vocabulary must be connected to the source topic and the mission scenario.

### Video Briefing Placeholder

A placeholder for a future short instructional briefing.

Use this format:

```text
Video Briefing Placeholder:
[Future video: brief description of what the cadet will see or hear.]
```

The placeholder should describe the intended briefing, not link to a missing asset.

### Writing Exercise

A written output the cadet would plausibly produce in maritime training or onboard communication.

Examples:

- Short report
- Log entry
- Calculation note
- Position statement
- Maintenance observation
- Cargo measurement summary

The writing exercise must not be a generic paragraph prompt.

### Practical Activity

A hands-on or decision-based action that demonstrates the topic.

Examples:

- Complete a report using given operational facts
- Calculate a measurement from provided data
- Select the correct technical view
- Plot or describe a vessel position
- Compare operational options and justify one

### Assessment

Assessment must evaluate performance, not isolated grammar.

The assessment should verify whether the cadet completed the operational objective accurately, clearly, and appropriately for the scenario.

Assessment may include language quality when relevant, but language must serve the mission outcome.

### XP Reward

XP remains simple for now.

Use a clear, small reward value such as:

```text
XP Reward: 10 XP
```

Do not create complex leveling, badges, ranks, or economy systems inside individual tasks.

### Quality Checklist

Every task must include a checklist confirming that it meets the standard.

Required checklist:

- Traces back to one official source topic.
- Does not invent academic topics.
- Feels like a maritime cadet mission.
- Requires the student to perform an operational action.
- Uses grammar or vocabulary as support, not as the final objective.
- Assesses performance in context.
- Uses a mission-style title.
- Keeps XP simple.

## Quality Rules

1. Every task must trace back to one official source topic.

4. The student must perform an operational action.
5. Grammar must not be the final objective.

## Bad Example vs Good Example

### Bad Example

```text
Practice 1.1: Presentarse a si mismo
```

Why it fails:

- It sounds like a classroom exercise.
- It does not define a maritime role.
- It does not create an operational situation.
- The likely objective is language practice instead of onboard performance.

### Good Example

```text
Mission 1.1: Report To The Chief Engineer
```

Why it works:

- It places the cadet in a maritime chain of command.
- It gives the student a clear operational audience.
- It can still use introductions and identity language, but the final action is reporting correctly.
- It supports academy training instead of generic English practice.

**Source Subject:** Maritime English.

**Mission Title:** Mission 1.1: Report To The Chief Engineer

**Mission Brief:** The cadet must report clearly to the chief engineer at the beginning of an engine room familiarization watch. The mission focuses on giving essential identity, duty, and readiness information in a professional maritime context.

**Operational Scenario:** You have just entered the engine control room for your assigned familiarization period. The chief engineer asks who you are, what duty you have been assigned, and whether you are ready to follow engine room safety instructions.

**Cadet Role:** Engine cadet reporting to the chief engineer.

**Operational Objective:** Deliver a short, complete self-report that identifies the cadet, states the assigned duty, and confirms readiness to follow instructions.

**Key Vocabulary:**

- Chief engineer
- Engine room
- Duty
- Watch
- Ready
- Safety instructions

**Video Briefing Placeholder:**
[Future video: chief engineer receives a new cadet in the engine control room and asks for a formal report.]

**Writing Exercise:** Write the short report you would give to the chief engineer. Include your name, cadet role, assigned duty, and readiness statement.

**Practical Activity:** Choose the most appropriate report from three options, then improve the selected report so it is complete, respectful, and operationally clear.

**Assessment:** The response is successful if the cadet identifies themselves, states the assigned duty, confirms readiness, uses an appropriate professional tone, and communicates clearly enough for the chief engineer to act on the information.

**XP Reward:** 10 XP
